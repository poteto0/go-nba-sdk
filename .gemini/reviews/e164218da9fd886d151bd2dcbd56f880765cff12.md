### Silent failure on invalid Proxy URL

**File:** `internal/http_client.go`

In `NewHttpClient`, the error from `url.Parse(config.ProxyUrl)` is ignored.

```go
if config.ProxyUrl != "" {
    proxyUrl, err := url.Parse(config.ProxyUrl)
    if err == nil {
        httpClient.Transport = &http.Transport{
            Proxy: http.ProxyURL(proxyUrl),
        }
    }
}
```

**Risk:** If the user provides a malformed URL, the client will be created **without the proxy** and without any warning. This is dangerous as the user might expect traffic to be proxied (e.g., for security or access control) but it will go direct.

**Recommendation:**
Since `NewHttpClient` does not return an error, you should consider:
1.  Changing the signature to `func NewHttpClient(...) (IHttpClient, error)`.
2.  Panicking on invalid configuration (if strict validation is desired at startup).
3.  At minimum, logging the error if the signature cannot be changed.

### Idiomatic Naming

**File:** `types/config.go`

The field `ProxyUrl` should be named `ProxyURL` to follow Go conventions (acronyms should be all caps, e.g., `ServeHTTP`, `ParseURL`).

```go
type GnsConfig struct {
    // ...
    ProxyURL string // instead of ProxyUrl
}
```

### Strict Error Checking in Tests

**File:** `internal/http_client_test.go`

In `Test_CreateHttpClient_WithProxy`, the error from `http.NewRequest` is ignored with `_`.

```go
req, _ := http.NewRequest("GET", "http://example.com", nil)
```

It is better to assert no error to ensure the test setup is valid:

```go
req, err := http.NewRequest("GET", "http://example.com", nil)
assert.NoError(t, err)
```
